# Codex Task Brief Compiler Policy

Complex harness changes, target rollouts, product-relevant changes, and bugfixes require a structured task brief. The brief captures goal, context, allowed scope, forbidden scope, done criteria, verification surface, risk surface, oracle requirement, rollback condition, runtime readiness claim, product code changed claim, expected files, and do-not-touch list.

Target rollout briefs must state harness-managed files only and must not claim product readiness.
