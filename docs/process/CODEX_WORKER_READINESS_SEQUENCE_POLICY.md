# Worker Readiness Sequence Policy

- Worker readiness sequence is schema and pure helper, receipt reconciliation, claim timeout retry, worker entrypoint, then staging no-tx evidence.
- Worker entrypoints must not precede state machine foundations.
- Runtime readiness requires staging no-tx or stronger oracle evidence.

Safe output only: no raw logs, raw diffs, secrets, endpoints, private paths, production data, or personal data. This policy adds no external dependency, LLM judge, MCP, browser, Playwright, AST parser, npm dependency, product command execution, product code change, or runtime readiness claim.
