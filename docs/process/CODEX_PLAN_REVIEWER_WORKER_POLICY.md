# Plan Reviewer Worker Policy

Product behavior plans are reviewed before implementation by `owner_reviewer`, `cutover_reviewer`, and `evidence_reviewer`.

The alignment result is `aligned`, `manual_required`, or `fail`. Wrong owner, missing cutover, weak evidence, forbidden scope, runtime readiness sweetening, mixed product/refactor scope, or unclear security boundary fails.
