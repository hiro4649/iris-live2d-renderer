# Codex Product PR Evidence Generator Policy

CODEX_QUALITY_HARNESS_FILE v1.0.5

Product PR evidence packs must include task mode, risk, changed files,
expected product files, local pre-push evidence, formal product evidence, safe
npm diagnostic, remote baseline, same-head remote quality gate, review
independence, readiness boundaries, limitations, and one next action.

Pre-push evidence is not remote pass. Pending-after-push is not remote pass.
Fixture pass is not runtime readiness. Browser smoke is not production
readiness.
