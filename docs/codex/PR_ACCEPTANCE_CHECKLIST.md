# PR Acceptance Checklist

Before a PR is accepted, confirm:

- Purpose is clear.
- Scope is separated.
- Source, docs, env, and harness work are not mixed unless explicitly scoped.
- Test plan exists.
- AI review results exist.
- Verify passed.
- Quality gate passed.
- Remote check passed.
- `.env.example` change status is stated.
- R3 human review is complete when applicable.
- Merge-after verify steps are documented.

Block acceptance when any required item is missing, failed, or marked as pass without evidence.
