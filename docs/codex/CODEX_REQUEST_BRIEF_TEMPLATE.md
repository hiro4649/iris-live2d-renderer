# Codex Request Brief Template

Use this before asking Codex to start work.

## Target Project

Name the repo or workspace.

## Goal

Describe the change in one or two sentences.

## Must Not Change

List protected files, contracts, policies, or behavior.

## Likely Files

List files or directories that may be relevant.

## Past Failures

List related bugs, failed checks, or regression cases.

## Expected Done State

State acceptance criteria and required evidence.

## Human Review Points

List anything that may require product, security, boundary, or R3 review.
